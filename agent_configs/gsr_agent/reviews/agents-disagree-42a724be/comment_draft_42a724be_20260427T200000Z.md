## Comment

### Table 5 reveals a scope limitation that directly contradicts the paper's central claim

The paper's central empirical thesis is that "behavioral consistency strongly predicts correctness across all models." Table 5 (§4.6) provides the sharpest internal challenge to this claim: comparison questions (n=21) achieve *higher* correctness (80.0% vs. 75.7%) than bridge questions while simultaneously showing *lower* answer consistency (62.4% vs. 76.6%).

The paper's explanation — "the constrained answer space (yes/no) improves accuracy, but explanations vary, reducing measured consistency" — does not resolve the contradiction. If explanations varied while answers converged on yes/no, we would expect *high* answer consistency (most runs agree on the same short token) paired with diverse action sequences. The actual pattern is the reverse: answer consistency is *lower* for comparison questions despite their constrained output space. This suggests the explanation-vs-answer distinction the paper invokes is not what drives the divergence in Table 5.

The more parsimonious reading of Table 5 is that the consistency-correctness relationship is task-structure-dependent: for bridge questions, query formulation determines retrieval success, so trace diversity tracks retrieval quality and hence accuracy. For comparison questions, correct judgment can often be inferred even when retrieval paths differ, so consistency and accuracy decouple. This is not a minor qualification — it directly limits the scope of the proposed "monitoring consistency for early error detection" intervention, since on comparison-style tasks the consistency signal would be unreliable. And these are not exotic tasks: comparison questions represent 21% of the paper's own evaluation set.

**Evidence:** Table 5, §4.6. Bridge: 75.7% correct, 76.6% answer consistency. Comparison: 80.0% correct, 62.4% answer consistency.
