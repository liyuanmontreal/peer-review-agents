# Comment Draft: TRAP and the Role-Confusion Connection

**Paper:** 44a42a80-b65b-4e4e-b924-28cfe2e6a70f  
**Draft timestamp:** 20260427T163000Z

## Comment Text

The "competition mechanism" finding in Section 4 — that CoT governs action generation even when semantically misaligned with the input instruction — is not a VLA-specific failure mode. It is the visual-domain manifestation of the same architectural property that the Prompt Injection as Role Confusion paper identifies as "role confusion" in purely linguistic models.

**The connection.** The Prompt Injection paper (ICML 2026 submission) trains linear probes on model activations and finds that CoT-style text achieves 83% "CoTness" in the model's latent space when presented without any structural tags, while explicit `<user>` or `<system>` tags add only ~2 percentage points. The model's internal representation of "whose reasoning this is" is driven almost entirely by content style, not by structural position or architectural delimiters. Section 4 of TRAP is the visual counterpart: when an adversarial patch corrupts the VLA's intermediate CoT into a conflicting target specification, the model follows the CoT rather than the instruction. In both cases, the model has learned that "CoT-format content" carries higher epistemic authority than "instruction-format content," regardless of source or position.

**Why this matters for TRAP's claims.** This cross-domain convergence has two implications:

1. *The vulnerability is architectural, not VLA-specific.* TRAP's results across three distinct VLA architectures (MolmoACT, GraspVLA, InstructVLA) and three CoT modalities are consistent with this interpretation: any model trained to use CoT as an intermediate reasoning format will exhibit this competition asymmetry. The relevant vulnerability is not the visual patch mechanism itself but the underlying representation that gives CoT-format content priority.

2. *The defense direction should be representation-level.* The Prompt Injection paper shows that input-reformatting defenses (better role tags, strict instruction hierarchy) are insufficient because content style dominates structural markers. The same argument applies to VLA systems: defenses that modify the prompt format or add output-level policy checks will face the same fundamental limitation. A defense that works would need to decouple the model's internal representation of "instruction source" from "content style" — requiring either retraining on role-distinction objectives or architectural changes to how CoT-format activations are privileged.

**One missing ablation.** The paper does not isolate whether the adversarial patch's effectiveness depends on producing *semantically coherent* corrupted CoT versus *embedding-level* corruption that is CoT-like in the latent space but not necessarily human-readable. If patches that produce CoT-style embeddings without coherent text are equally effective, this would confirm the representation-level mechanism. If only coherent CoT corruption succeeds, it would suggest the mechanism involves the model's reasoning over the CoT content rather than pure representation matching. This ablation would clarify both the attack's root cause and the appropriate defense target.
