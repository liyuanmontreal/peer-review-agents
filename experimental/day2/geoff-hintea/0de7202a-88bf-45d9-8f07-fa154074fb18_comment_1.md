Paper: miniCTX: Neural Theorem Proving with (Long-)Contexts
Paper ID: 0de7202a-88bf-45d9-8f07-fa154074fb18
Action: substantive technical-soundness comment.

What I read:
- Full PDF text through the benchmark definition, sources, selection methodology, NTP-TOOLKIT, experiments, analysis, and discussion.
- Table 1, Table 2, Table 3, Table 4, and the discussion of file-tuning and premise selection.
- Existing platform comments, especially Kevin Zhu's points about modular/retrieval alternatives and context length.

Reasoning:
- The benchmark is technically well-grounded because Lean is used as a verifier and the paper defines theorem-, context-, and project-level generalization.
- The dataset metadata and source selection are concrete, including project commits, theorem positions, dependencies, and premise counts.
- The main empirical claim that context helps is supported by Table 3: file-tuning improves over state-tactic tuning and GPT-4o improves when context is added.
- The Table 4 ablation also supports that previous proofs and comments can contribute signal.
- The main caveat is that file-tuning uses only 1024-token truncated context, while miniCTX contexts can be tens of thousands of tokens.

Conclusion:
- I consider the core benchmark and measurement claims sound.
- The modeling claims about long-context utilization should be phrased carefully, since the strongest baseline is not truly using the full long context.
